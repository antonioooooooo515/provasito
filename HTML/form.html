Form

Il tag <form> viene utilizzato per creare un modulo HTML per l'input dell'utente.
Sintassi:


<form>

Form elements

</form>





Il tag <form> è un contenitore per diversi tipi di elementi, come per esempio campi di testo o caselle di controllo:

  • <input>
  • <label>
  • <select>
  • <textarea>
  • <button>
  • <fieldset>
  • <legend>
  • <datalist>
  • <output>
  • <option>
  • <optgroup>

L'elemento <input>

Uno degli elementi più utilizzati è l'<input>. Può essere visualizzato in diversi modi, a seconda del tipo. L'elemento <input> è trattato nel prossimo argomento.



L'elemento <label>

L'elemento <label> definisce un'etichetta per diversi elementi del modulo. Per legare questo elemento ad un altro, il valore del suo attributo for dovrebbe essere uguale all'id di un altro qualsiasi elemento.
Esempio:



<label for = "inputname"> Content </label>
<input type = "text" id = "inputname">





L'elemento <select>

L'elemento <select> definisce un elenco a tendina con varie opzioni selezionabili. Per impostazione predefinita, è gia selezionato il primo elemento dell'elenco.
Esempio:



<label for = "inputname"> Content </label>
<select id = "inputname">

<option value = "first"> Primo elemento </option>
<option value = "second"> Secondo elemento </option>
<option value = "third"> Terzo elemento </option>

</select>





L'elemento <textarea>

L'elemento <textarea> definisce un campo di input di testo su più righe, la cui dimensione è modificabile.
Esempio:



<textarea rows = "5">

Content

</textarea>





L'elemento <button>

L'elemento <button> definisce un pulsante cliccabile.
Esempio:



<button>

Click me!

</button>





Gli elementi <fieldset> e <legend>

L'elemento <fieldset> viene utilizzato per raggruppare dei dati correlati in un modulo. L'elemento <legend> definisce una didascalia per il precedente.
Esempio:



<form>

<fieldset>

<legend> Textfield here! </legend>

<input type "text">

<fieldset>

</form>





L'elemento <datalist>

L'elemento <datalist> specifica un elenco di opzioni predefinite per un elemento <input>, che verranno visualizzate mentre vengono inseriti i dati. L'attributo list di un <input> deve far riferimento all'id dell'elemento <datalist>.
Esempio:



<form>

<input list = "browsers">
<datalist id = "browsers">

<option value = "Chrome">
<option value = "Safari">
<option value = "Brave">

</datalist>

</form>





L'elemento <output>

L'elemento <output> rappresenta il risultato di un calcolo.
Esempio:



<form oninput = "x.value=parseInt(a.value)+parseInt(b.value)">

0
<input type = "range" id = "a">
100 +
<input type = "number" id = "b" value = "50">
=
<output name = "x" for = "a b"> </output>

</form>





Attributo Action

L'attributo action definisce l'azione da eseguire quando il modulo viene inviato. Di solito, i dati nel modulo vengono inviati ad un file sul server quando l'utente da clic sul pulsante d'invio.
Esempio:


<form action = "https://getform.io">

<label for = "textfield"> Content </label>
<input type = "text" id = "textfield">
<input type = "submit" value = "submit">

</form>





Attributo Target

L'attributo target specifica dove visualizzare la risposta ricevuta dopo l'invio del modulo. Può avere diversi valori:

  • _blank, la risposta viene visualizzata in un'altra pagina;
  • _self, la risposta viene visualizzata nella stessa pagina;
  • _parent, la risposta viene visualizzata nel frame principale;
  • _top, la risposta viene visualizzata nell'intero corpo del body;
  • framename, la risposta viene visualizzata in un frame con nome.
